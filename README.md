# my-resume-new
personal resume website
### Revert 說明
在本次作業中，我使用了 git revert 回復某個 commit，原因如下：

為了練習版本控制，我先在 main 分支上故意加入一個假錯誤（錯誤文字 / 註解），並 commit。  
隨後，我使用 `git revert` 產生一個新的 commit 來回復這個假錯誤，使本地與遠端的 main 分支恢復到正確版本。  
這樣可以確保在不破壞現有正確程式碼的情況下，練習 revert 操作並留下清楚的版本歷史紀錄。
### A/B 版本說明
在 Skills 區塊，我實作了兩個版本：

- **版本 A：進度條 (Progress Bars)** → 已合併到 main 分支並採用  
  原因：進度條可以清楚呈現技能熟練度，視覺效果直覺，符合簡歷網站的設計需求。

- **版本 B：標籤雲 (Tags Cloud)** → 保留分支未合併  
  原因：雖然標籤雲有趣，但在簡歷網站上資訊呈現不夠直觀，容易讓使用者難以快速了解技能熟練程度，因此選擇不採用。
